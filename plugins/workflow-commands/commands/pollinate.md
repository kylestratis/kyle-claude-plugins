# Pollinate

**Arguments:** `$ARGUMENTS`

Use your Skill tool to engage the `pollinate` skill. Follow it exactly as written.

This command analyzes a source feature, maps it to target conventions, and generates a design document for porting. It does NOT write code directly — it feeds into the standard `/plan` → `/execute` → `/verify` pipeline.
